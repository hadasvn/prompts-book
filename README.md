# ספר פרומפטים חכם

אפליקציית React לוקאלית לגמרי (בלי שרת) — הופכת את 20 הפרומפטים הקבועים לכלי עבודה חי:
פרופיל עסק גלובלי, מילוי אוטומטי של placeholders, ניתוח תיאור עסק, ייבוא וסיווג פרומפטים
חיצוניים, ושמירה לקובץ מקומי / לתיקיית Drive.

כל הנתונים (פרופיל, פרומפטים שמורים, הגדרות) נשמרים ב-`localStorage` של הדפדפן — אין שרת ואין
מסד נתונים חיצוני.

## חשוב — למה הקוד רץ מ-C:\ ולא ישירות מתוך Drive
Google Drive (הכונן G:\) הוא כונן וירטואלי שלא מסתדר טוב עם `node_modules` (אלפי קבצים קטנים) —
ניסיון להתקין תלויות ישירות בתוך תיקיית ה-Drive גרם לשיבוש קבצים. לכן:

- **הפיתוח בפועל** רץ מהעותק המקומי: `C:\Users\hadas\dev\prompts-book`
- **קוד המקור מסונכרן** (בלי `node_modules`) גם לתיקיית ה-Drive, כדי שיהיה נגיש דרך Git/לצפייה.
- כדי להריץ את האפליקציה תמיד עובדים מול העותק ב-`C:\Users\hadas\dev\prompts-book`, לא מתוך Drive.

## הרצה

### להדס — הפעלה יומיומית
לחיצה כפולה על **"הפעלת האפליקציה.bat"** בתיקייה הזו. הוא מפעיל את העותק המקומי
(`C:\Users\hadas\dev\prompts-book`) ופותח את הדפדפן אוטומטית.

### לשיתוף עם מישהו אחר
קובץ ה-bat הרגיל לא יעבוד אצל אף אחד אחר — הוא מצביע לנתיב קבוע במחשב של הדס בלבד.
כדי לשתף:
1. מעתיקים/דוחסים (zip) את כל התיקייה הזו (בלי צורך למחוק כלום — אין כאן `node_modules`)
2. שולחים לחבר/ה
3. הם מחלצים את הקובץ **למקום רגיל במחשב שלהם** (לא בתוך Google Drive/OneDrive/Dropbox משלהם)
4. לוחצים לחיצה כפולה על **"הפעלה למחשב אחר (לשיתוף).bat"** — הוא בודק אם יש Node.js מותקן,
   מתקין תלויות אוטומטית בפעם הראשונה (לוקח דקה), ואז מפעיל את האפליקציה. יש ל-Node.js
   דרישה מוקדמת: מותקן מ-https://nodejs.org

### הרצה ידנית (למי שמכיר טרמינל)
```
npm install   # פעם אחת, או אחרי הוספת תלויות חדשות
npm run dev   # מריץ את האפליקציה על http://localhost:5173
```
